# Bibfoot


![The logo of bibfoot, showing an animal footprint.](./figures/bibfoot.png)


A tool to plug bibtex references into markdown in way that is rendered properly in github/hugo 

## Purpose

When writing notes in markdown, be they for a literate program or a wiki, it would be nice to have a proper pipeline that extracts citations from the markdown file (just like a `\cite` in LaTeX), retrieves the relevant information from bib files using bibtex, and then adds the properly formatted references in a final `References` section of the markdown file. `bibfoot` does just that.


### Usage

After installing it (see below), bibfoot is used as follows. The files are also given in the `tests` folder.

First, you need to write a markdown file ([test.md](tests/test.md)) such as the following one, 
```md
# A very interesting note


## First Section

Which cites a very interesting paper[^AC-BonPerTia19] (obviously picked uniformly at random).

```
and the correspdonding [bibtex file](./tests/biblio.bib) (or just take it from [DBLP](https://dblp.org):
```bib
@inproceedings{AC:BonPerTia19,
  author       = {Xavier Bonnetain and
                  L{\'{e}}o Perrin and
                  Shizhu Tian},
  editor       = {Steven D. Galbraith and
                  Shiho Moriai},
  title        = {Anomalies and Vector Space Search: Tools for S-Box Analysis},
  booktitle    = {Advances in Cryptology - {ASIACRYPT} 2019 - 25th International Conference
                  on the Theory and Application of Cryptology and Information Security,
                  Kobe, Japan, December 8-12, 2019, Proceedings, Part {I}},
  series       = {Lecture Notes in Computer Science},
  volume       = {11921},
  pages        = {196--223},
  publisher    = {Springer},
  year         = {2019},
  url          = {https://doi.org/10.1007/978-3-030-34578-5\_8},
  doi          = {10.1007/978-3-030-34578-5\_8},
  timestamp    = {Tue, 26 Nov 2019 14:13:27 +0100},
  biburl       = {https://dblp.org/rec/conf/asiacrypt/BonnetainPT19.bib},
  bibsource    = {dblp computer science bibliography, https://dblp.org}
}
```


Now, let's add the proper reference. For this, first run `bibfoot` with the `extract` argument as follows.

```sh
bibfoot extract test.md biblio.bib
```

If you have multiple bibliography files, just add them all in a space-separated sequence after the markdown file. This creates the `.aux` file that bibtex needs, so you can just run

```sh
bibtex test
```

Then, to put the resulting info (which `bibtex` stores in a `.bbl` file) back in your markdown file, simply run

```sh
bibfoot rewrite test.md
```

It will look for a section called `References` and replace its content (**and everything that comes after**) with the data generated by `bibtex`. It will do so in such a way that the references in the text, which correspond to the citation syntax of markdown, work as you would expect.

The result looks like this (see also the [actual file](./tests/test-after.md)):

```md
# A very interesting note


## First Section

Which cites a very interesting paper[^AC-BonPerTia19] (obviously picked uniformly at random).

## References

[^AC-BonPerTia19]: Xavier Bonnetain, Léo Perrin, and Shizhu Tian. Anomalies and vector space search: Tools for s-box analysis. In Steven D. Galbraith and Shiho Moriai, editors, Advances in Cryptology - ASIACRYPT 2019 - 25th International Conference on the Theory and Application of Cryptology and Information Security, Kobe, Japan, December 8-12, 2019, Proceedings, Part I, volume 11921 of Lecture Notes in Computer Science, pages 196–223. Springer, 2019.
```

### Caveats

- As said above, the `References` section is assumed to be the last, and its content is savagely erased. Make sure you haven't written anything before calling `bibfoot rewrite`!

- The way github renders markdown footnotes does not allow the identifiers to be arbitray. In particular, it doesn't like the `:` symbol. As a consequence, `bibfoot` replaces all `-` in the markdown citation with `:` in the `.aux` file (and vice versa when rewriting the markdown). This works well for my intended use case, which is to use the bibtex files from [cryptobib](https://cryptobib.di.ens.fr/); however, it might be more annoying if you e.g. keep the keys from DBLP.

## Installation

This is a basic python project, so running the following in your terminal should work out-of-the-box. It will download this program, its dependency (namely [pylatexenc](https://pylatexenc.readthedocs.io/en/latest/)), create an executable called `bibfoot`, and put it somewhere in your `$PATH`.

```sh
pip install git+https://github.com/lpp-crypto/bibfoot.git
```
